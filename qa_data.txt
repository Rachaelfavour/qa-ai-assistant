=== AUTHENTICATION MODULE ===

LOGIN - POSITIVE SCENARIOS
- Verify user can login with valid username and password
- Verify login works using remembered session
- Verify login works across different browsers

LOGIN - NEGATIVE SCENARIOS
- Verify error message for invalid password
- Verify error message for invalid username
- Verify validation for empty username/password fields
- Verify account lock after multiple failed attempts

LOGIN - EDGE CASES (POSITIVE)
- Verify system handles maximum input length correctly
- Verify login works with boundary values

LOGIN - EDGE CASES (NEGATIVE)
- Enter very long username/password beyond limit and verify rejection
- Enter special characters and verify system stability
- Attempt SQL injection and verify it is blocked

LOGIN - E2E FLOW
- Navigate to login page
- Enter valid credentials
- Click login
- Verify dashboard is displayed
- Verify session is maintained


LOGOUT - POSITIVE SCENARIOS
- Verify user can successfully log out using logout button
- Verify user is redirected to login page after logout
- Verify session is terminated after logout

LOGOUT - NEGATIVE SCENARIOS
- Attempt to access protected pages after logout and verify access is denied
- Attempt API request after logout and verify unauthorized response (401)

LOGOUT - SESSION MANAGEMENT
- Verify session expires after inactivity timeout
- Verify user is logged out automatically after timeout
- Verify session is invalidated across all tabs

LOGOUT - MULTI-DEVICE / MULTI-TAB
- Login on multiple devices and logout from one device – verify other sessions
- Logout from one tab and verify session behavior on another tab

LOGOUT - EDGE CASES
- Logout during active transaction and verify system behavior
- Logout during network interruption and verify consistency
- Click logout multiple times and verify no system error
